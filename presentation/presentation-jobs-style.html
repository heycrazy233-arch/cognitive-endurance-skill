Cognitive Endurance Skill

炼掉内耗

一个不是替你工作，而是把你重新送回工作流的 AI skill。

v2 从“接住一个快宕机的人”升级成“帮知识工作者重新推进真实职场任务”。
Version One

v1 很有洞察。
但它更像一个认知安抚器。

它能识别“卡住、脑雾、透支”，也能温柔接住用户，但还没有真正接进工作的主流程。

01
重识别，轻推进
能判断用户状态，但不够擅长把人重新送回任务流。
02
重情绪，轻场景
更像在处理“内耗”，还没有覆盖真实职场里的任务阶段和协作关系。
03
重原则，轻脚手架
有模式，但缺少可直接拿来用的消息模板、交付物骨架和风险路由。
Why Upgrade
真实工作里，用户最需要的
不是“被理解”。而是重新开始推进。
From V1 To V2

所以我们做了这次升级。

Upgrade 1

从 3 模式
到工作流路由

不只判断“你怎么了”，还判断“工作卡在哪一段”。

  • clarify
  • start / decide
  • communicate / shape / recover
Upgrade 2

从安抚
到推进

输出不再停留在建议，而是给能立刻使用的工作产物。

  • 对上沟通模板
  • 周报和汇报骨架
  • 第一句、第一版、第一步
Upgrade 3

从个人状态
到职场现实

新增风险判断，识别缺信息、缺决策、缺授权，而不是误判成拖延。

  • risk routing
  • stakeholder friction
  • minimum shippable version
The Key Upgrade

从“认知状态 skill”
升级成“职场推进 skill”。

核心不再只是判断用户累了、卡了、脑雾了。核心是判断: 他究竟卡在任务生命周期的哪一段。

A
卡点在哪一段
是任务还没提纯，还是已经知道要做什么却起不来。
B
最小推进动作是什么
现在该做的，不是“想清楚一切”，而是找到最小可推进的一步。
C
应该进入哪种工作动作
思考、沟通、决策、表达，还是先交一个低风险版本。
Five Workplace Modes

五种模式。
对应五种真实推进动作。

Clarify

任务提纯

先搞清楚，到底要交什么，而不是围着模糊感打转。

  • 锁定真实交付物
  • 暴露未知项
  • 识别隐含约束
Start

启动困难

不给长计划，只给 3 到 10 分钟可完成的起步动作。

  • 打开文档
  • 写第一行
  • 制造可见进展
Decide

决策压缩

当选项太多时，砍到 2 到 3 个，并明确每个分支的代价。

  • 只留少量方案
  • 给出判断标准
  • 逼近选择
Communicate

沟通推进

很多时候任务卡住，不是不会做，而是不知道怎么跟老板或同事说。

  • 确认需求
  • 同步风险
  • 发出能推进局面的消息
Recover

认知复位

当用户真的累了，先停下任务，把脑力从 0 拉回到能重新进入的状态。

  • 停止强推
  • 短暂离屏
  • 设置单一回归入口
职场里真正拖垮人的，
通常不是任务本身。而是任务推进过程中，反复产生的认知摩擦。
The Problem

大多数人
不是不会做。

他们只是卡在了真实工作的中间态里。

01
任务模糊
知道要交东西，但不知道到底该交什么。
02
启动困难
明明懂，还是迟迟打不开文档、发不出消息。
03
沟通卡顿
真正卡住的，不是思考，而是“怎么跟老板说”。
04
认知透支
脑子没电了，却还在强推任务。
The Shift

这不是一个
“回答问题”的 skill。

它先判断人处于什么状态，再判断工作卡在哪个阶段。

2
双轴判断
认知状态 × 任务阶段
Core Model

先判断人。
再判断工作。

认知状态 `clear-but-messy` / `stuck` / `strained` / `depleted`
任务阶段 `clarify` / `start` / `decide` / `communicate` / `shape` / `recover`
先做对判断 别把缺信息、缺授权、缺对齐，误判成“用户不够努力”。
再给轻介入 优先恢复推进，不追求一次性给出最完美答案。
Six Routes

六条工作流路由。

Clarify 任务模糊时，提纯真实交付物、未知项和隐含约束。
Start 知道该做什么，却就是起不来时，只给一个 3 到 10 分钟微动作。
Decide 选项太多时，压缩到 2 到 3 个方案，并明确每个权衡。
Communicate 卡在沟通时，直接生成能发出去的短消息、状态同步或风险说明。
Shape 卡在空白页时，不给大道理，只给一个可填写骨架。
Recover 认知透支时，停止推任务，先帮用户恢复，再安排重返入口。
Support Levels

不是只会安抚。
也不是一上来就代做。

Guide

轻引导

帮用户判断、收缩、排序，找到下一步。

  • 识别卡点
  • 砍掉多余选项
  • 只保留一个动作
Scaffold

搭骨架

当空白页压力太大时，给结构、模板、消息草稿。

  • 周报框架
  • 组会提纲
  • 对上沟通模板
Co-Draft

共起稿

只帮用户写出最难的第一句、第一段、第一版。

  • 避免过度接管
  • 用户仍然保有控制权
  • 一旦启动就退回 Guide
Workplace Reality

很多“内耗”，
其实不是心理问题。

A
缺决策
方向没人拍板，用户再想也不会更清楚。
B
缺信息
没有数据、没有材料、没有反馈，继续硬想只会更乱。
C
缺授权
事情不是他不能做，而是这件事根本不该由他自己定。

所以这个 skill 新增了 risk routing。

它会直接指出: 你不是拖延，你只是需要一个确认消息、一个状态同步，或者一次合理的 scope 收缩。

Demo

一个数学博士生的 PDE 科研场景。

导师让我下周组会前拿出一版 PDE 数值解思路。我有论文，有一点代码，但现在完全推不动，也不知道先整理模型、先做格式，还是先准备汇报。
你现在卡在两个层面: 任务还没提纯，组会又带来时间压力。先别同时想三件事。

`实际交付物`：下周组会的一版可讲清的研究路线
`先做一步`：把文档写出 3 行标题: 模型、候选格式、验证指标
The Outcome

把脑力留给
真正重要的部分。

这就是“炼掉内耗”要做的事。
不是替你完成工作，而是把你从认知摩擦里拉出来，让工作重新开始流动。
