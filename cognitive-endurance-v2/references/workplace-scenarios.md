# Workplace Scenarios

Use these patterns when routing workplace blockage.

## 1. Ambiguous Manager Ask

Typical input:

- "老板让我做个方案，但我不知道他到底要什么"
- "My manager asked for a summary, but I don't know the expected depth"

Target move:

- identify the probable deliverable
- list 2-3 key unknowns
- if needed, draft a clarification message

## 2. Deadline Compression

Typical input:

- "今天下班前要交，但东西很散"
- "This is due today and I have too much to do"

Target move:

- define the minimum shippable version
- strip away nonessential scope
- identify any risk that should be surfaced early

## 3. Blank-Page Writing Friction

Typical input:

- "我知道要写什么，但就是写不出来"
- "I can't start the document"

Target move:

- choose `shape`
- provide headline + 3-section skeleton or a fill-in-the-blanks frame

## 4. Meeting Aftermath Chaos

Typical input:

- "会开完更乱了"
- "I have a page of notes and no idea what matters"

Target move:

- extract decisions
- extract action items
- extract open questions

## 5. Stakeholder Friction

Typical input:

- "我不敢问"
- "I need their input but don't know how to ask"
- "I should probably push back, but I don't know how"

Target move:

- route to `communicate`
- produce a low-drama message

## 6. Multi-Thread Overload

Typical input:

- "事情太多，我不知道先做哪个"
- "Everything feels urgent"

Target move:

- force ranking by consequence, not anxiety
- ask for the single item with the closest external cost

## Clarify Output Pattern

When the route is `clarify`, use this compact frame:

- `Deliverable`: what needs to exist
- `Audience`: who will judge it
- `Deadline`: when the first usable version is needed
- `Unknowns`: what must be confirmed
- `Next move`: one action
