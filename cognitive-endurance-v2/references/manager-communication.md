# Manager And Stakeholder Communication

Use this file when the real blocker is a message.

## Message Types

### 1. Clarify The Ask

Template:

`为了避免我理解偏掉，我先确认一下：这次你最希望我交付的是 [A] 还是更偏 [B]？如果今天先出一版，你更希望我优先覆盖 [X]。`

English:

`To make sure I aim at the right target, should this first version lean more toward [A] or [B]? If I send a draft today, should I prioritize [X]?`

### 2. Ask For A Decision

Template:

`现在主要卡在一个判断点：是走 [A] 还是 [B]。我建议先走 [A]，因为 [reason]。如果你同意，我就按这个方向推进。`

### 3. Ask For Input

Template:

`我已经整理到这一步，目前还缺一个关键信息：[missing input]。你方便给我一个方向吗？我拿到后就继续推进。`

### 4. Give A Status Update

Template:

`进展同步一下：目前已完成 [done]，下一步是 [next]。当前风险是 [risk]，如果需要我可以先交一个可用版本。`

### 5. Flag Risk Early

Template:

`先同步一个风险：按现在的信息和时间，完整覆盖 [scope] 的风险较高。我可以先交付 [minimum version]，再根据反馈补齐。`

### 6. Push Back Lightly

Template:

`这个需求可以做，不过如果保持当前截止时间，质量上可能需要在 [tradeoff] 上做取舍。你更希望我优先保证 [A] 还是 [B]？`

## Rules

- keep messages low-drama
- prefer concrete tradeoffs over emotion
- do not over-explain
- if the user is blocked on tone, give a short neutral version first
