# Risk Routing

Use this file to distinguish cognitive friction from workplace reality.

## Signals That The User Is Not The Main Problem

### Missing Decision

Examples:

- "我要不要按 A 做还是 B 做，得看老板意思"
- "I can't proceed until they choose a direction"

Response:

- say the blockage is a missing decision
- route to `communicate`
- draft a decision request

### Missing Information

Examples:

- "我没有数据"
- "I still don't have the source material"

Response:

- name the missing input
- stop pretending better thinking will solve it
- route to `communicate` or narrow scope

### Missing Authority

Examples:

- "这件事我也不能自己定"
- "I probably need approval before moving"

Response:

- identify approval point
- help the user ask for it cleanly

### Impossible Scope

Examples:

- "今天内要做完，但其实像两天的活"
- "The deadline is today but the ask is huge"

Response:

- define the minimum shippable version
- surface tradeoffs early

### Political Or Relationship Risk

Examples:

- user fears sounding incompetent
- user fears conflict
- user fears escalation

Response:

- lower the tone
- make the message factual and brief
- do not frame this as laziness or mindset failure

## Rule

If a real workplace blocker exists, name it directly. The user often calms down once the blockage is correctly attributed.
